Preguntas a responder:

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red global de computadoras interconectadas. Es decir, la infraestructura. Mientras que WWW es un sistema de información para proporcionar acceso a la información como paginas web.

¿Cuáles son las partes de un URL?

Protocol: EL protocolo de comunicación, este genera un request a un servidor que responde con algún resource. Ejemplos: HTTP, HTTPS (versión segura), FTP (para transferencia de archivos entre clientes y un servidor en una red), SMTP (transmisión por mail).

Domain: El nombre del dominio, es decir, la dirección de la página web. Ejemplo: www.google.com

Path: La ruta del recurso que se quiere obtener. Ejemplo: /search?q=hola

Query: Los parámetros que se le pasan al servidor. Ejemplo: ?q=hola

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET = Leer

HEAD = Similar al GET, peror devuelve los encabezados

PUT = Actualizar

PATCH = Actualizar parcialmente

POST = Subir

DELETE = Eliminar

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar el método POST, ya que este método envía los datos al servidor para que sean procesados.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET, ya que las solicitudes GET solo deben recuperar datos.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que la solicitud fue exitosa. No ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Si ya que este error significa que el recurso no fue encontrado en el servidor.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si, ya que este error significa que hubo un error interno en el servidor.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que el atributo ya no es parte de la especificación de HTML. Por ejemplo: font, center, big, strike, tt, frame, dir.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 tiene un mejor soporte para multimedia, tiene una mejor semántica, tiene una mejor conectividad y una mejor experiencia de usuario.

Input, select, textarea, button, fieldset, label, legend, datalist, output.
